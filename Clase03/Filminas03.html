¿Por Qué Necesitamos Funciones?

Fundamentos

Una función es un bloque de código reutilizable diseñado para realizar una tarea específica, evitando copiar y pegar la misma lógica.

  • Reutilizar código: Escribes la lógica una vez y la usas mil veces.
  • Modularizar: Divides un problema grande en piezas pequeñas y manejables.
  • Abstraer: Usas una función como enviarEmail() sin recordar cada línea interna cada vez que la llamas.

Declaración de Función

Sintaxis Core

La función declarada es la manera más tradicional y directa de crear una función: un anuncio oficial en tu código.

  • Nombre obligatorio: Al ser una declaración independiente, debe tener un nombre para poder referenciarla.
  • Hoisting (Elevación): JavaScript "mueve" virtualmente estas declaraciones al principio del código antes de ejecutarlo.
  • Claridad: Son fáciles de identificar como "herramientas" disponibles en todo el archivo.
// Declaración de función function saludar(nombre) { console.log("Hola, " + nombre + "!") }

Expresión de Función

Ciudadanos de Primera Clase

Las funciones pueden tratarse como cualquier otro valor y asignarse a una variable: a esto lo llamamos función expresada.

  • Pueden ser anónimas: Se conocen por el nombre de la variable, no tienen nombre propio.
  • No tienen Hoisting: No puedes llamar a esta función antes de la línea donde la definiste.
// Expresión de función const despedirse = function(nombre) { console.log("Adiós, " + nombre) };

Hoisting: El Truco de Magia

#FindTheBug

Las funciones declaradas se elevan al inicio del archivo; las expresadas no. ¿Por qué falla la segunda?

  • Llamar a una función declarada antes de escribirla funciona, aunque no es recomendado: dificulta la lectura y organización del código.
  • Llamar a una expresión de función antes de su línea de definición lanza un error.
// Esto funciona perfectamente presentarse() function presentarse() { console.log("Hola, soy una función declarada.") } // Esto causará un ERROR: cocinar(); const cocinar = function() { console.log("Cocinando...") } // Uncaught ReferenceError: Cannot access 'cocinar' before initialization

Parámetros y Argumentos: El Input

Analogía

Imagina una cafetera: la ranura es el parámetro, el café real que depositas hoy es el argumento.

  • Parámetros: Variables que definimos en la declaración de la función. Son como "huecos" o etiquetas.
  • Argumentos: Los valores reales que le pasamos a la función cuando la llamamos.
function saludar(nombre) { // 'nombre' es el parámetro console.log("Hola, " + nombre) } saludar("Julieta") // "Julieta" es el argumento

El Retorno de Valores: El Output

Palabra Clave return

Para que una función nos devuelva un resultado que podamos guardar en una variable, usamos return.

  • Regla de oro: Cuando el motor de JavaScript encuentra un return, la función se detiene inmediatamente.
  • Nada de lo que escribas debajo de un return se ejecutará.
function duplicar(numero) { return numero * 2 // Envía el resultado hacia afuera } let miResultado = duplicar(5) // miResultado ahora vale 10

Función Flecha: La Evolución

ES6

Las funciones flecha ofrecen una sintaxis más concisa para funciones anónimas, con retorno implícito.

  • Sin retorno explícito: Si una función no tiene return, JavaScript hace que devuelva undefined por defecto (efecto secundario o "side effect").
  • Retorno implícito: En una sola línea, la función flecha retorna el resultado automáticamente, ideal para callbacks.
// Función flecha con retorno implícito const suma = (a, b) => a + b console.log(suma(5, 3)) // Salida: 8

El Scope Global: El "Clima" del Código

Fundamentos

El Scope es el conjunto de reglas que definen dónde puede ser alcanzada una variable dentro de tu código.

  • Scope Global: El nivel más externo del programa. Toda variable declarada fuera de una función pertenece a este ámbito.
  • Disponibilidad total: Una variable global puede ser leída y modificada por cualquier función escrita después de su declaración.
// Variable en Global Scope const nombreAplicacion = "Mi Super App" function mostrarBienvenida() { // Podemos acceder a la variable global desde aquí console.log("Bienvenido a " + nombreAplicacion) } mostrarBienvenida() // Imprime: Bienvenido a Mi Super App

El Peligro de los Globales

Conflictos de Nombres

Si tienes un programa grande con muchas variables globales, podrías intentar declarar dos cosas con el mismo nombre accidentalmente, generando una "contaminación del scope global".

Dificultad de Mantenimiento

Si una variable global cambia su valor, es difícil rastrear qué parte del código la modificó, ya que cualquier función pudo haberlo hecho.

Scope Local: Tu "Habitación Privada"

#FindTheBug

Las variables declaradas dentro de una función solo existen "dentro" de ese espacio. ¿Por qué falla la segunda línea?

  • Scope de Función: Cuando declaras una variable dentro de una función, solo esa función puede verla.
function calcularImpuesto(monto) { let tasa = 0.21 // Variable LOCAL a la función return monto * tasa } console.log(calcularImpuesto(100)) // Funciona: 21 console.log(tasa) // ERROR: tasa is not defined

Scope de Bloque: let y const

ES6

Un bloque es cualquier cosa rodeada por llaves {}, como un if, un for o un while.

  • Las variables declaradas con let o const dentro de un par de llaves solo son visibles dentro de esas llaves.
if (true) { const saludo = "Hola, estoy atrapado en el bloque" console.log(saludo) // Funciona } console.log(saludo) // ERROR: saludo no está definido

Lexical Scope y Shadowing

Jerarquía del Scope

JavaScript utiliza Lexical Scope: las funciones anidadas acceden a las variables de su scope exterior, pero no al revés.

  • Shadowing (Sombreado): Una variable local con el mismo nombre que una global la "sombrea" dentro de su scope, sin modificar la global por fuera.
const color = "azul" // Global function cambiarColor() { const color = "rojo" // Local (Shadowing) console.log(color) // Imprime "rojo" } cambiarColor() console.log(color) // Imprime "azul"

Síntesis y Resumen del Scope

Scope Global

Variables declaradas fuera de cualquier función o bloque. Accesibles desde cualquier parte del programa.

Scope Local / Función

Variables declaradas dentro de una función. Solo existen mientras la función se ejecuta.

Scope de Bloque

Variables (let, const) declaradas dentro de { }. Solo existen dentro de esas llaves.

Transparencia

Las funciones internas pueden ver variables de sus padres, pero los padres no pueden ver las variables de sus hijos.
